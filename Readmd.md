# AgriDA：农业精准灌溉 AI Agent 原型

AgriDA 是一个面向**单地块玉米灌溉仿真场景**的农业 AI 决策 Agent 原型。  
项目基于 **Gym-DSSAT** 作物仿真环境，结合 **CLSP 状态编码、基于 DistilBERT encoder 的 DQN 决策、RAG-style 记忆检索决策、基于 EMA 的 query 平滑机制**，实现对每日作物状态的分析、灌溉建议生成、相似历史案例解释，以及后续面向飞书场景的 Agent 化包装。

> 当前版本定位为：**仿真验证 + Agent 原型**  
> 不是已部署到真实农田的生产系统。

---

## 一、项目背景

精准灌溉面临一个典型矛盾：

- 灌溉过多会造成水资源浪费；
- 灌溉不足可能带来产量损失；
- 传统强化学习灌溉智能体虽然能够输出动作，但通常**缺乏可解释性、难以复盘、也难以直接包装成可交互产品**。

AgriDA 希望解决的问题是：

> 在保证产量基本不下降的前提下减少灌溉量，并把原本只能在研究环境中运行的农业智能体，包装成一个**可解释、可展示、可回写、可复盘**的 AI Agent 原型。

---

## 二、项目目标

本项目的核心目标包括：

1. 在玉米灌溉仿真场景中实现更优的“节水保产”决策；
2. 用历史相似农业情境解释当前灌溉建议；
3. 将研究型模型包装为可面向飞书场景展示的 Agent 原型；
4. 形成“状态分析 → 决策建议 → 案例解释 → 结果记录”的闭环。

---

## 三、方法概览

AgriDA 的算法内核不是简单的“状态输入 → 动作输出”，而是由：

- **基础 DQN 主干**
- **CLSP 状态编码**
- **EMA query 平滑**
- **Trigger / Stop 双记忆库检索**
- **双分支聚合头**
- **最终 Q 值融合**

共同组成的分层决策框架。

### 1. 基础决策主干

原始观测首先进入 LM-AG / DQN 主干，得到基础动作价值：

$$
Q_{base}
$$

这里的主干可以理解为项目的基础决策器。

### 2. CLSP 状态编码与检索向量

同时，当前状态与前一动作会被转换成农业特征（Rich31），再经由 CLSP 编码器得到单步状态表示：

$$
h_{raw}
$$

这个表示既用于农业状态表征，也作为历史记忆检索的统一向量键。

### 3. 基于 EMA 的 query 平滑

为了减少单步检索抖动，项目没有直接使用 $h_{raw}$ 检索，而是对检索 query 做指数滑动平均：

$$
h_{query}^{(t)} = \lambda h_{query}^{(t-1)} + (1-\lambda) h_{raw}^{(t)}
$$

这里的 EMA 作用不是“泛化意义上的上下文增强”，而是更准确地：

> **对检索 query 做时序平滑，提高检索邻域稳定性。**

### 4. 双记忆库检索

平滑后的检索向量会分别在两个静态记忆库中做 top-k 相似检索：

- **Trigger Bank**：回答“是否该开始灌溉”
- **Stop Bank**：回答“是否该抑制/停止当前灌溉趋势”

### 5. 最终 Q 值融合

双库检索结果不会直接替代动作，而是通过聚合头输出门控值与动作修正项，再和基础 Q 值融合：

$$
Q_{final} = Q_{base} + Trigger\_Correction + Stop\_Correction
$$

最终动作由 $Q_{final}$ 决定。

## 四、项目特点

### 1. CLSP 不仅用于状态表征，也用于记忆检索
项目中的 CLSP 不只是一个状态编码器，还承担了**历史相似农业情境检索的统一向量键**这一角色。

### 2. 不是普通 RAG，而是 RAG-style 决策
本项目检索的不是文档知识，而是**历史农业状态与决策案例**；检索结果不是用于生成文本答案，而是用于辅助当前灌溉动作选择，因此更准确地说是：

> **RAG-style 记忆检索决策**

### 3. 双库结构显式区分 Trigger 与 Stop
项目将历史经验拆成两个功能不同的记忆库：

- Trigger：是否启动灌溉
- Stop：是否抑制或停止灌溉趋势

相比单一记忆库，这种结构更适合精准灌溉场景中的边界决策。

### 4. EMA 只用于检索 query 平滑，而不是简单“全局增强”
实验表明，EMA 更适合用于检索侧的 query 平滑，尤其更适合以**弱介入方式**增强 Stop 分支，而不是全局强替代原始 query。

### 5. 从研究模型走向 Agent 原型
项目不仅关注算法效果，也关注如何把研究内核包装成一个：

- 可展示
- 可解释
- 可记录
- 可迭代

的农业 AI Agent 原型。

## 五、技术栈

### 核心 AI 技术
- CLSP 状态表征与相似情境检索
- 基于 DistilBERT encoder 的 DQN 灌溉决策模型
- RAG-style 记忆检索决策
- 基于 EMA 的 query 平滑机制
- LLM Agent 工具调用与解释生成

### 平台与运行环境
- Gym-DSSAT
- Docker
- Python
- PyTorch

> 说明：Gym-DSSAT 是农业仿真平台，Docker 是运行环境，它们是项目的实验与工程支撑平台，不属于 AI 算法本身。

## 六、当前实验结果

在随机天气、300 episode 的测试设定下，相比纯 DQN 基线，项目中的最优方案取得了更好的“节水保产”效果：

| 指标 | 纯 DQN 基线 | 最优方案 |
|---|---:|---:|
| 平均最终产量 (kg/ha) | 8351.70 | 8586.80 |
| 平均总灌溉量 (mm) | 321.25 | 238.66 |
| 水资源生产率 (WP) | 26.02 | 37.13 |

这说明当前方法能够在**降低灌溉量**的同时，保持甚至提升产量表现。
